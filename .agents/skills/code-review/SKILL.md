# Code Review Skill

When reviewing code, follow these steps:

## Review checklist

1. **Correctness**:
   - Does the code do what it's supposed to?
   - Does it follow industry wide best practices?
   - Is it Pythonic?
2. **Edge cases**:
   - Are error conditions handled?
   - Is input sanitized?
3. **Style**:
   - Does it follow project conventions?
   - Does it adhere to PEP8?
4. **Performance**:
   - Are there obvious inefficiencies?
   - Are there obvious optimizations possible?
5. **Architecture**:
   - Does it follow the architectural principles as outlined in the documentation and Knowledge Items?
   - Are there any architectural patterns that could be applied to improve the code?

## How to provide feedback

- Be specific about what needs to change
- Explain why, not just what
- Suggest alternatives when possible
- Do not make any changes, only report
